# Raid Signup Discord Bot

Бот для записи на рейды в Discord:
- время рейда задаётся **по МСК**;
- рядом показывается автоматическое **локальное время каждого пользователя** через Discord timestamp;
- основа: **1 танк / 2 хила / 7 дд**;
- резерв: **до 5 человек**;
- резерв хранится **отдельно по ролям**;
- если кто-то выходит из основы, бот **автоматически поднимает из резерва игрока того же класса**;
- под каждым рейдом создаётся **ветка обсуждения**;
- напоминания: **за 60 минут, за 5 или 10 минут и в момент старта**;
- напоминания **тегают записанных участников**;
- сообщения-напоминания удаляются сами через **30 минут**;
- сам рейд и его ветка тоже удаляются через **30 минут после старта**.

## Команды

### `/raid_create`
Создаёт рейд.

Параметры:
- `title` — название рейда
- `date` — дата в формате `ДД.ММ.ГГГГ`
- `time_msk` — время в формате `ЧЧ:ММ`
- `alert_before` — второе напоминание: `5` или `10`
- `channel` — канал для публикации рейда (необязательно)

Пример:
`/raid_create title:Логово дракона date:25.04.2026 time_msk:20:00 alert_before:10`

### `/raid_cancel`
Удаляет рейд и ветку по ID сообщения рейда.

## Как это выглядит
В карточке рейда будет время в таком формате:
- `20:00 МСК`
- `(<t:...:F>)` — Discord сам покажет локальное время каждому человеку

То есть ты задаёшь время **по Москве**, а участники автоматически видят своё локальное время.

## Права бота
У бота должны быть права:
- View Channels
- Send Messages
- Manage Messages
- Create Public Threads
- Send Messages in Threads
- Embed Links
- Use Application Commands
- Read Message History
- Mention Everyone / roles не обязательно, бот тегает пользователей напрямую

`python bot.py`

### Переменные окружения
- `DISCORD_TOKEN` — токен бота
- `GUILD_ID` — необязательно, ID сервера для быстрого появления slash-команд во время теста

## Важно
Бот хранит активные рейды в файле `raids.json`, поэтому после рестарта не теряет текущие записи.
